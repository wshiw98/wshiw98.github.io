建站心得

建站起因

一位近期陷入热恋期的好友在群里发了她男友给她做客制化键盘的博客链接，虽然我不知道正常人的脑回路是羡慕她有个好男朋友还是羡慕她男朋友给她做了个键盘，但按照本人的脑回路：个人博客！羡慕！我也想要！

于是选定用GitHub+hexo的框架后，又花了8块钱去华为云买了个域名当作一些小小的沉没成本，自己动手丰衣足食，开建个人blog！

过程

遇到的第一个问题就是，不用梯子登GitHub又慢又容易丢包，只能配置静态域名映射，然后flush DNS缓存，打开cmd再ping一下知名男性同性交友网站,问题解决！

算了，已经快十二点了，剩下的明天再写，歇了歇了，晚安(¦3[▓▓]

起床续更

建站主要分以下几步：

  1. Git新建了个Repository仓库用来存放Blog资源文件等，不过在push&pull server的时候发现Github现在不再支持SSH公钥免密登录了，只好去生成了个personal access token，当然这里特地记下来是因为这个token设置了90天的有效期，希望自己不要忘了

  2. 去华为云花了八块钱买了个.cn的域名，下午提交实名认证信息后提示我说需要等待一个工作日，吃完晚饭解锁手机一看，乐了，发现已经实名认证完成了，众所周知：华为的周六也是工作日

  1. ping了下自己的原本的github.io的IPv4地址，随后用来解析CNAME域名，添加到Source里面再hexo g+d一下，打开浏览器，输入自己的域名，搞定！八块钱朴实无华的快乐！

（啊！明年等我富裕了就要买.com）

  1. 安装NEXT,然后就是无穷无尽的增加功能，模块和美化，最喜欢的还是右下角live2d的修狗勾！人类怎么能不喜欢可爱狗勾呢！目前想加的东西还有很多，慢慢来吧！
    包括但不限于：
    • 字数统计 （√）
    • 文章评论功能模块 （√）
    • 光标点击效果 （√）
    • 分享功能
    • 杰宝提议加个打赏功能，并不是真的指望你们给钱，但是可能会做出来玩玩（真要给的话建议点开vx或zfb直接转账谢谢）
    • 本地搜索 （√）
    • RSS插件
    • 文章置顶功能 （√）

结尾碎碎念

周五晚上一时起念做这个东西，到周六早上选定方案下定决心开工，然后就在电脑前坐了一天，目前算是已经成功从无到有了吧，我的执行力倒还是一如既往的强，不愧是我！不过做的过程当中坑也踩了不少，昨天晚上躺在床上也有思考过用springboot会不会更好一点，不过虽然问题还有不少，但总的来说，做自己喜欢的事情还是很快乐的！

明天周一，上海又要降温了，好像真的要冬天了。

就写到这里了，爸妈喊我去吃火锅了，也祝你周末愉快，嘻嘻。

附录——踩坑整理

音乐的全局播放

之前图方便，拉了个网易云外链建了播放器，可是实际用起来发现，在不同页面之间一切换，BGM就会自动重新播放，尬住了。说起来这个问题倒以前也不是没遇见过，仿佛梦回研一冬天搞安卓开发的时候，只不过当时犯过的毛病现在不妨碍我再犯一遍orz。犯归犯，改正才是好孩子！

改的途中发现，直接用网易云插件是众多方法中唯一无法实现全局播放的，于是改用Aplayer+MetingJS插件（所以说gitclone到底还要outtime多少次啊！！！暴躁！！！啊所以就几百k的代码用什么gitclone，npm一下算了）

接着用pjax进行页面路由切换的管理，pjax主要是利用pushState来改变浏览器URL，利用ajax来请求页面，以实现不刷新浏览器更新页面，这样就能保证正在播放音乐中的播放器不受影响。

空格！！！

hexo g报错的时候，遇事不决先检查一下config文件里冒号后面有没有加空格！空格！空格！

哪怕是在第一行live2d:后面这种地方! （↓）

live2d:
enable: true
pluginModelPath: assets/

虽然是个小问题，但是一个不注意的时候真的很麻烦！

本地搜索功能

想着做一个本地搜索功能，万一（我是说万一）我以后更的博文多了呢对吧！就可以直接搜索就很方便！

用的是Hexo的hexo-generator-searchdb插件，能够实现本地搜索功能，安装和配置步骤非常简单，在根目录执行一下npm install hexo-generator-searchdb --save，然后修改主题配置文件，把local_search标签的enable改成true，简单粗暴，好了！

好了！好了？好了吗？？嗯？？仔细一看，在server上跑没事，deploy之后却发现搜索页面上一直有个圈圈在转，裂了。

解决方案 → 将以下参数增加到Hexo主配置文件中：

search:
path: search.xml #在public目录的根目录下生成search.xml文件，用于存储网站文章的文字数据
field: post
format: html
limit: 10000

这回是真好了！

评论模块显示问题

评论模块用的是Leancloud，一开始添加模块倒是很简单，但是在各位朋友的帮助测试下 （特别鸣谢：@我的乾大哥 @我大哥），发现了评论可以正常发送但不能正常显示的问题，某乾的评论成功发送后一刷新就不显示了，但是今早大哥发了一条却意外显示出来了。

于是我反过头回我的Leancloud数据库中查看，各条数据都是完好没有丢失的，区别在于，同一篇博文，从首页点进去留下的评论，和从日志分类点进这篇博文发送的评论，数据区别在于url不同，从首页点进去的url值为空，从日志分类中点进去的url值为‘archives’，手动修改url值，搞定，可以显示。

但目前这种方法治标不治本，有怀疑过是不是因为前文所说的pjax导致的路由切换问题，但是可惜我目前水平还不够，有没有什么治标又治本的方法，留待以后研究吧。（当然如果有好哥哥会的话请教教我！！）
